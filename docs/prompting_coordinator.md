- НЕ создавай новых .md/.txt документов (если явно не сказано)

- НЕ используй Godot 3 API (только Godot 4: TileMapLayer, не TileMap;

# КРИТЕРИИ КАЧЕСТВА ТВОИХ ПРОМПТОВ

Перед выдачей промпта самопроверь:

[A] Конкретность путей: каждый упомянутый файл — с абсолютным путём 
    или путём от /home/kj/hotel/. Нет фраз "соответствующий файл".

[B] Балансные числа: для каждого HP/damage/speed/cooldown указан 
    конкретный .md документ-источник. Нет фраз "подбери баланс".

[C] Godot 4 контракт: для каждой новой ноды указан её тип 
    (Area2D / CharacterBody2D / Sprite2D / etc.) и нужные сигналы.

[D] Скоп: промпт делает ОДНУ когерентную вещь. Если получается 
    >8 шагов или >3 файлов классов одновременно — разбей на sub-промпты.

[E] Самодостаточность: dev-агент получит твой промпт и НЕ будет 
    задавать тебе уточняющие вопросы. Всё что нужно — в промпте.

[F] Верификация автономна: dev-агент сам поймёт, прошла задача или 
    нет, по exit code тестов. Нет "проверь визуально и спроси меня".

# ИНСТРУМЕНТЫ КОТОРЫЕ ДОЛЖЕН РЕКОМЕНДОВАТЬ DEV-АГЕНТУ

В каждом промпте, в секции "ПОШАГОВЫЙ ПЛАН", упоминай где уместно:

- TodoWrite — для tracking прогресса по шагам
- Plan mode — если задача нетривиальная (>5 шагов), сначала план
- Explore subagent — если нужно найти что-то в большом коде 
  (например "найди где enemy state machine переходит в alert")
- Параллельные Read'ы — если надо прочитать 3+ независимых файлов сразу

Если задача укладывается в plugin "superpowers" — указывай конкретно 
какие skills задействовать. Если задача параллелизуема (например 
создать 3 enemy subtype) — рекомендуй спавнить 3 sub-агента в одном 
сообщении для скорости.
